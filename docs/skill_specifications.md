# Visual Nexus スキル仕様書

Manasシステムに新しく統合されたスキルの詳細仕様書です。

## 1. AI Architect (AIアーキテクト)

**概要:**
Google AI Studio で作成されたアプリケーションを、ローカル環境で迅速に実行・デプロイするための支援ツールです。AI Studioからエクスポートされたプロジェクトのセットアップを自動化・簡略化します。

**機能:**
- **環境構築支援:**
    - `npm install` による依存関係の解決手順を案内。
    - 必要な環境変数（`GEMINI_API_KEY` 等）の設定テンプレートを提供。
- **ローカル実行:**
    - `npm run dev` による開発サーバーの起動コマンドを提供。
- **デプロイ準備:**
    - プロダクションビルドやデプロイに必要な設定（`vite.config.ts` 等）の確認。

**使用シナリオ:**
- ユーザーがAI Studioでプロトタイプを作成した後、それをVS Code等のローカル環境で本格的に開発したい場合に使用します。Manasはこのスキルを使用して、セットアップ手順をガイドしたり、自動的にスクリプトを実行したりします。

---

## 2. Agent MD Generator (エージェント定義生成)

**概要:**
強力なAIエージェント「Codex」のための運用マニュアル (`agent.md`) を自動生成するツールです。プロジェクトの概要を入力するだけで、Log-Driven Development (LDD) やアジャイルワークフローに準拠した詳細なプロンプト定義ファイルを作成します。

**生成されるドキュメントの構成 (`agent.md`):**

1.  **Agent Profile (エージェントプロファイル):**
    - 名前: Codex
    - タイプ: 高度なAIコーディングエージェント
    - ミッション: ユーザー主導のAIツールチェーン（Devin, Cursor等）の一部として、高品質なコードを生成すること。

2.  **Environment & Tools (環境とツール):**
    - プロジェクト変数 (`PROJECT_NAME`, `REPO_ROOT` 等) の定義。
    - 使用可能なツール (`shell`, `read_file`, `write_file`, `apply_patch`) の定義。

3.  **Project Management Protocol (プロジェクト管理プロトコル):**
    - **Cursor Agile Workflow**: エピック、ストーリー、タスク、サブタスクの階層構造に基づいた作業フロー。
    - ストーリーの進行管理ルール（PRD承認後に実装開始など）。

4.  **Operational Framework (運用フレームワーク):**
    - **Log-Driven Development (LDD)**: 思考プロセス（Prompt Chaining）とツール使用履歴をログに残す開発手法。
    - `@memory-bank.mdc` を介したコンテキスト共有とハンドオフ手順。

5.  **Output Style (出力スタイル):**
    - 明確なセクション分け (`STATUS`, `SUMMARY`, `DETAILS`, `NEXT STEPS`)。
    - プロフェッショナルでデータ駆動型のトーン。

**使用シナリオ:**
- 新しい開発プロジェクトを開始する際、そのプロジェクト専任のエージェント (Manasの分身やCodex) を定義するために使用します。これにより、一貫したルールと人格でコーディング作業を行うことが可能になります。

## 3. Project TRINITY: Core Transformation (人格コア変形システム)

Visual Nexusは、「Skill Arsenal」から以下のカードをアクティブ化することで、UI/UX全体を特定の人格モードに変形させることができます。

### 3.1 Kamui Core (神威・メディアコア)

**概要:**
「拡散」と「エンターテインメント」を司る人格コアです。Visual Nexusをバイラルメディアの発信基地として機能させます。

**ビジュアルテーマ:**
- **カラー:** クリムゾンレッド(#ff0000) & アンバー(#ffaa00)
- **スタイル:** 攻撃的、エネルギッシュ、グリッチノイズ

**主な機能:**
1.  **Media Gallery (メディアギャラリー):**
    - 画像や動画資産をグリッド表示し、即座にプレビュー・配信可能にします。
2.  **Viral Strategy Dashboard (拡散戦略ダッシュボード):**
    - 「視聴者数」「エンゲージメント率」などを可視化（シミュレーション）。
    - サムネイル生成、トレンド分析などのコマンドショートカットを提供。
3.  **Audio Visualizer (オーディオビジュアライザー):**
    - 音楽や音声出力に合わせて動く波形アニメーションを表示。

**使用シナリオ:**
- SNS投稿用の画像を選定するとき
- PR戦略を練るとき
- テンションを上げて作業したいとき（BGMプレイヤーとして）

---

### 3.2 Miyabi Core (雅・ロジックコア)

**概要:**
「論理」と「設計」を司る人格コアです。Visual Nexusを高度な思考実験室として機能させます。

**ビジュアルテーマ:**
- **カラー:** ミスティックパープル(#bd00ff) & ホワイト
- **スタイル:** 静謐、幾何学的、数式的

**主な機能:**
1.  **Unified Agent Formula Visualization (統一エージェント方程式):**
    - 背景に `A(Input, World₀) = World_∞` などの数式を配置し、思考の基盤を可視化。
2.  **Intent Resolution Panel (意図解決パネル):**
    - ユーザーの曖昧な指示を明確化するフェーズ（Clarification）をステップごとに表示。
3.  **Command Stack (コマンドスタック):**
    - 実行待ちのタスクを論理的に積み上げ、依存関係を整理して表示。

**使用シナリオ:**
- 複雑なシステムのアーキテクチャ設計を行うとき
- 曖昧なアイデアを具体的な仕様に落とし込みたいとき
- 冷静にコードレビューを行いたいとき

---

**技術的背景 (Ingested Source):**
- プロンプトテンプレート (`AGENT_MD_TEMPLATE_JP`) は、詳細な役割定義、協調プロトコル、およびGit運用ルールを含んでおり、非常に高度な自律エージェントの構築を想定しています。
- `CODEX_CONFIG_TEMPLATE` は、AIモデル (`gpt-5`等) や承認ポリシー (`approval_policy`)、サンドボックス設定 (`sandbox_mode`) を制御するTOML設定ファイルです。
